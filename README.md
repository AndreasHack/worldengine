<p>You can generate the data for your own world, including a number of images (heigthmap, biomes, etc.).</p>
<p>For example:</p>
<div class="highlight highlight-source-shell"><pre>worldengine world -s 1 -n seed1</pre></div>
<p>Worlds are generated using plate simulations, erosion, rain shadows, Holdridge life zones model and plenty of other phenomenons.</p>
<p>The created world can be used for simulating the evolution of a civilization (see project <a href="https://github.com/ftomassetti/civs">civs</a>).</p>
<p>It is also possible to generate additional maps, for example an ancient looking map:</p>
<div class="highlight highlight-source-shell"><pre>worldengine ancient_map -w seed1.world</pre></div>

<p>We would love to see WorldEngine used together with other tools. Worlds generated by WorldEngine can be loaded into python applications using WorldEngine itself as a library. Java applications can instead use <a href="https://github.com/Mindwerks/worldengine-java">WorldEngine-Java</a>, a java library to load WorldEngine files.</p>
<p>Worlds can be saved using the protobuf format or hdf5, for which there are libraries in several languages. We keep working on supporting more formats and always interested in ways to improve interoperability.</p>

<h1><a id="user-content-algorithm" class="anchor" href="#algorithm" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Algorithm</h1>
<p>The world generation algorithm goes through different phases:</p>
<ul>
<li>plates simulation: it is the best way to get proper mountain chains. For this <a href="https://github.com/Mindwerks/pyplatec">pyplatec</a> is used</li>
<li>noise techniques are used at different steps</li>
<li>precipitations are calculated considering latitude and rain shadow effects</li>
<li>erosion is calculated</li>
<li>humidity in each zone is calculated</li>
<li>terrain permeability is calculated</li>
<li>biome is calculated using the <a href="http://en.wikipedia.org/wiki/Holdridge_life_zones">Holdridge life zones</a> model</li>
</ul>

<h3><a id="user-content-windows" class="anchor" href="#windows" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Windows</h3>
<p>Turning your Windows into a developer environment is a long and drawn out process.
I'll try to keep this as short as possible and to the point.</p>
<p>Remember, be consistent if you are either win32 or win64 and everything you download
and install is either one or the other, but not both.</p>
<p>You'll want to install msysgit: <a href="https://msysgit.github.io/">https://msysgit.github.io/</a> which will get you
a Linux like environment. After that, clone the repo and install Python 2.7 for
windows: <a href="https://www.python.org/downloads/windows/">https://www.python.org/downloads/windows/</a> This will get you also pip
which is required for the rest. You'll first need to pip install virtualenv.</p>
<p>The layout is a bit different than in Linux.</p>
<div class="highlight highlight-source-shell"><pre>virtualenv venv
venv/Scripts/pip install -r requirements.txt</pre></div>
<p>Numpy install will fail, so you'll need download a pre-compiled wheel file and
install it with pip. <a href="http://www.lfd.uci.edu/%7Egohlke/pythonlibs/#numpy">http://www.lfd.uci.edu/~gohlke/pythonlibs/#numpy</a>
numpyâ1.9.2+mklâcp27ânoneâwin_amd64.whl
Pick one for your arch, 32 or 64.</p>
<div class="highlight highlight-source-shell"><pre>venv/Scripts/pip install ../numpyâ1.9.2+mklâcp27ânoneâwin_amd64.whl</pre></div>
<p>Next step is to get pywin32 which are win32api hooks for python, when downloading,
you'll need to pick either 32 or 64-bit otherwise it won't work. You'll also
install it via pip. <a href="http://sourceforge.net/projects/pywin32/files/pywin32/">http://sourceforge.net/projects/pywin32/files/pywin32/</a></p>
<div class="highlight highlight-source-shell"><pre>venv/Scripts/pip install ../pywin32-219.win-amd64-py2.7.exe</pre></div>
<p>The last step is to get pyinstaller installed and this can be tricky
because as of right now, we have to use a specific revision that "good-enough".
The issue is being tracked here: <a href="https://github.com/pyinstaller/pyinstaller/issues/1291">https://github.com/pyinstaller/pyinstaller/issues/1291</a></p>
<div class="highlight highlight-source-shell"><pre>venv/Scripts/pip install git+https://github.com/pyinstaller/pyinstaller.git@67610f2
venv/Scripts/pyinstaller --clean --console -F -n worldengine worldengine/__main__.py</pre></div>
